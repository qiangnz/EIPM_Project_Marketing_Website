Project Verifone EIPM

About the Project


The Client

Verifone, Inc. is one of the largest FinTech companies worldwide and processes 10.3 billion transactions in over 150 countries.

Verifone owns and operates the largest independent terminal and payment gateway in New Zealand - EFTPOS.

They power payments of over 50,000 New Zealand businesses, big and small.


Scenario

Verifone processes approximately 500 million transactions every year in New Zealand. These transactions are traditionally terminal centric. However, since the need for fast and secure online payment services is growing, Verifone now supports alternative payment methods such as QR code solutions and e-commerce transactions.

Currently, Verifone is launching a new global product that will support e-commerce transactions: E-commerce APIs that Verifone's third-party merchants could use. New Zealand is the first market that they are going to launch this new product.


Solution

The Project Verifone EIPM focused on the testing and the performance analysis of Verifone's API to maximise its value for the Client by developing an e-commerce website and integrating it with Verifone's Hosted Payment Page API, and performing Performance Testing on it.

Goals

There are two key deliverables of this project

  • 01Plan, build and test the creation of an e-commerce store

    The project's first goal is to plan, design, and develop an E-commerce website that can integrate with Verifone’s API. This website will simulate a merchant website that will make use of Verifone’s hosted payment page.

  • 02Performance Testing of Verifone’s Hosted Payment Page API

    The second goal is to test the capabilities and limitations of Verifone API. This will include performance testing and load testing to produce enough helpful data that the client can use.

 

Products

Website

The development of the website is the primary product that helped the team achieve the requirements for goal one.

The client was able to gather valuable feedback regarding website development based on the API specifications. Due to the website's customisability, Verifone will be able to use it to test their other APIs as well.



Performance Testing Dashboard

The second goal of the project was to performance test Verifone’s API and analyse the results.

A series of performance tests with varying load sizes and times were done in order to generate an average performance level for the API. Later on, the load was incremented in order to identify any degradation of performance. Then, results were analysed and the dashboard was created. The dashboard was used to present the result and its analysis to the client.
